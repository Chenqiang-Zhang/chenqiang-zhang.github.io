这是一个已经迁移到更适合长期维护的 Hexo 源码结构的博客站点。

## 现在的改造重点

- 文章使用 Markdown 编写，后续新增内容更加容易
- 主题和样式可以独立维护，不再依赖一堆手写静态 HTML
- 通过 GitHub Actions 自动部署到 GitHub Pages

## 你接下来可以这样做

1. 在 `source/_posts` 下新增新的文章
2. 通过 `npm run server` 本地预览
3. 执行 `npm run build` 生成静态站点

## 旧内容说明

你原来那批生成好的静态页面仍然保留在仓库中，作为历史内容备份；新的维护方式将围绕 Markdown 源文件展开。
